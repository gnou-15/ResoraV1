# Resora

### Description
Resora is a secure, interactive resume builder and benchmarking platform that evaluates resume content across target professions (IT, Healthcare, Education, and Management). It features zero-knowledge client-side encryption, real-time interactive resume scoring, and an automated GCash payment integration utilizing an SMS gateway.

---

## 🛠️ Technologies
*   **Frontend:** React 19, Vite 8, JavaScript (ES6+), Vanilla CSS
*   **Backend:** Python 3, FastAPI, Uvicorn, Pydantic
*   **Database & Auth:** Supabase (PostgreSQL), Row Level Security (RLS) policies, PL/pgSQL
*   **Security & Encryption:** CryptoJS (AES-256 & SHA-256)
*   **APIs & Third-Party:** Web3Forms API (Serverless Email submissions), Phone SMS Gateway webhook

---

## 🌟 Features
1.  **Zero-Knowledge Client-Side Encryption:** Auto-encrypts resume data locally using AES-256 before syncing to Supabase, ensuring host servers cannot read personal information.
2.  **Instant Scoring & Keywords Audit:** Evaluates resumes locally based on impact action verbs and quantitative data metrics.
3.  **Smart Metrics Optimizer:** Recommends dynamically contextualized bullet point improvements tailored to 4 different industry paths.
4.  **Interactive Parallax Mascot (Peeking Monster):** An SVG mascot that tracks cursor movements in real-time with responsive mood swings.
5.  **GCash Payment Hook:** Webhook integrations parsing SMS receipt logs automatically via database RPC procedures to manage trial limitations and upgrade licenses.
6.  **Genie Modal Transitions:** High-fidelity UI animations calculating coordinates relative to active buttons dynamically.

2.  **Zero-Knowledge Security:** Implemented cryptographic keys generated by combining individual user IDs and custom salt hashes so client data remains private.

4.  **Database Automation:** Created SMS routing functions within Supabase to handle automated payment verification workflows.

## 🧠 What I Learned

🔐 **Data Security & Cryptography:**
*   **Zero-Knowledge Encryption**: I integrated the `crypto-js` library to AES-256 encrypt user resume data on the client side using keys derived from a combined SHA-256 hash of a master pepper and user UUID. This guarantees that user resumes are never stored in cleartext, preventing data leaks at the database level.

📡 **Database Automation & Gateways:**
*   **PostgreSQL Webhooks & PL/pgSQL**: I wrote database stored procedures and RPCs in PostgreSQL to handle incoming webhooks from an SMS gateway. The system parses GCash texts with regex matching to automatically approve and manage user subscription tiers.

⚡ **Modern API Validation:**
*   **FastAPI & Pydantic Data Matching**: I used FastAPI to construct a lightweight REST API server and Pydantic models to mirror frontend JS resume data schemas. This ensures robust type validation, schema enforcement, and rapid backend processing.

📊 **Heuristic Analysis Algorithms:**
*   **Local Resume Parsing**: I built analytical scoring heuristics checking resume sections for key metrics (using regex filters for numbers, percentages, and currencies) and action verbs to grade alignments without expensive third-party AI models.

🎨 **Premium UI Engineering & Motion:**
*   **Dynamic Coordinate Calculations**: I computed element spatial vectors using React’s `getBoundingClientRect` API to construct smooth "Genie" transitions that scale and travel towards the origin layout on modal actions.

🛡️ **Screenshot Protection & Focus Control:**
*   **Keyboard & Screen Intercepts**: I implemented system-level visibility and keyboard event listeners (`keydown`, `blur`, `visibilitychange`) to automatically blur the preview canvas when `PrintScreen` or screen snipping hotkeys are detected, securing watermarked content.

## 🚀 How to Run the Project

### Prerequisites
*   Node.js (v18+)
*   Python (3.9+)
*   Supabase Account

### 1. Database Setup
1.  Log in to your Supabase Console.
2.  Open the SQL Editor and execute the schema definitions inside [database_setup.sql](file:///c:/Users/mapan/Documents/School%20Things/Practice%20Webs/ThirdWeb/random-web/backend/database_setup.sql) to provision tables, security rules, and RPC verification functions.

### 2. Backend Installation
1.  Open a terminal inside the `/backend` folder.
2.  Create a virtual environment and install dependencies:
    ```bash
    python -m venv .venv
    # Windows:
    .venv\Scripts\activate
    pip install -r requirements.txt
    ```
3.  Launch the API server:
    ```bash
    uvicorn main:app --reload
    ```

### 3. Frontend Installation
1.  Open a terminal inside the `/frontend` folder.
2.  Create a `.env` file based on `.env.example` and set your variables:
    ```env
    VITE_SUPABASE_URL=https://your-project.supabase.co
    VITE_SUPABASE_ANON_KEY=your-anon-public-key
    VITE_WEB3FORMS_ACCESS_KEY=your-web3forms-key
    ```
3.  Install dependencies and start the dev server:
    ```bash
    npm install
    npm run dev
    ```
